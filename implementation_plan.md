# ART&CRAFT 文化遗产网站 — 开发计划（基于 Figma Page 3）

## 项目背景

ART&CRAFT 是一个高端中国传统文化艺术数字展示平台。**本项目的所有开发将仅基于 Figma 设计稿的中 Page 3 最左侧的 Frame 内容进行**，完全不涉及 Page 1 和 Page 2，专注于呈现各种长篇历史文化专题及对应的藏品页面。

---

## 核心架构与技术栈（User Decisions Confirmed）

根据确认：
1. **基础框架**：选用 **Next.js (App Router) + TypeScript**，搭配 **Vanilla CSS (CSS Modules)** 进行精确的样式还原，保持代码纯净并贴近原生开发。
2. **内容管理系统 (CMS)**：采用 **Sanity**（或 Strapi）作为数据源，允许管理人员在不接触代码的情况下自由添加/修改后续文物数据，普通访客在前端网站只能浏览（只读）。
3. **部署方案**：
   - **前端**：**Zeabur**（支持国内直连加速）或通过 Vercel 并绑定免备案的香港服务器 CDN，确保中国内地及香港能够流畅且无阻碍地访问。
   - **CMS**：与前端一体化部署或托管至 Sanity Cloud。
4. **多语言支持**：引入 `next-intl` 或客户端 `opencc-js`（简繁转换库），在 Header 添加入口，实现整个站点的**简体中文 / 繁体中文**一键无缝切换。

## Figma Page 3 内容结构（56 个 Frame 全景梳理）

基于对 Page 3 图层结构的全面拉网式排查，确认有 56 个核心 Frame。我们将其归类为五种页面模板：

| 帧 ID | 类型 | 描述 |

| `D-LS-Y-001`, `D-LS-J-001`, `D-LS-J-002`, `D-LS-S-001`, `D-LS-S-001-G01` | 分类列表页 | 按不同文化主题划分的衍生列表视图 |

### 2. G-系列：长篇文化专题（宏大叙事、历史考证）
采用超长条卷轴式阅读设计。
| 帧 ID | 类型 | 描述 |
|--------|------|------|
| `G - 书画` | 长篇专题 | 丝绸之路上的纹样传播与书画艺术 |

## 阶段开发路径

### Phase 1: 基础设施搭建 (Infrastructure)
- 用 `create-next-app` 初始化项目（选用 App Router + TS）。
- 配置 CSS Modules 与全局变量（黑白灰色阶、繁简两套无衬线中英文字体栈）。
- 引入简繁转换拦截器（在 Provider 层实现中文切换）。
- 搭建 Sanity/Strapi CMS 后台，建立 `Artifact`, `Essay`, `Craft`, `List` Schema 以供上传图文素材。

### Phase 2: 全局 UI 组件化 (Components)
- 头尾及导航栏（带有全局搜索图标，取消原生登录注册的后台功能转而作纯展示或链接回 CMS）。
- 面包屑导航及圆角实体标签（Tag Component）。
- 长图/富文本解析组件（CMS Markdown -> HTML）。

### Phase 3: 页面模板化与响应式映射 (Templates)
1. `app/page.tsx` 映射 **Frontpage**。
2. `app/[category]/[id]/page.tsx` 构建通用详情模板，依据传入 CMS 数据动态判断应用哪种版式：
   - 模式一：**长图卷轴式 (G-系列)**（平滑滚动监听与章节高亮）。
   - 模式二：**两栏结构式 (D-系列)**（固定左侧文本滚动 / 粘性布局高清文物图展示）。
   - 模式三：**操作工序式 (CRA-系列)**（瀑布流或网格图库展示）。
3. 动态列表页提取 `D-LS*` 系列模板。

### Phase 4: 预览部署与联调 (Deploy)
- 连通部署至适合两地访问的 PaaS 平台。
- 在 CMS 中录入 Page 3 其中几个代表性帧（如 `G - 书画`, `D-QH-G01`, `CRA-001`, `D-LS`）的内容数据，完成线上真机验证。
